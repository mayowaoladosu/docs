## Identity and access in Nubis

Nubis IAM gives organizations one model for human access and machine access. The same control plane covers members, roles, service accounts, API keys, SSO configuration, and audit visibility.

## Member access

- Invite members into the organization with role-based access from the start.
- Review membership and invitation state in one place.
- Keep human access tied to organization-level policy instead of sharing credentials.

## Roles and permissions

Nubis supports:

- **System roles** such as Owner, Admin, and Member.
- **Custom roles** for organizations that need tighter separation of duties.
- **Fine-grained permissions** across compute, networking, storage, billing, IAM, and automation surfaces.

## Service accounts

Create non-human identities for CI/CD, bots, internal tooling, and controlled automation.

- Bind service accounts to roles instead of handing out broad user credentials.
- Use API keys for programmatic access.
- Rotate or revoke machine credentials without disturbing teammate access.

## SSO integration

Organizations that need centralized identity can configure SSO settings in IAM and verify the setup before rolling it out broadly.

## Audit posture

IAM changes matter most when you can see them later. Pair role changes, API key management, and service account usage with audit-log review and least-privilege defaults.
