## One control plane for modern infrastructure

Nubis gives startups and operator-led teams one place to run instances, databases, Kubernetes, storage, networking, domains, billing, and team access. The documentation follows the same model as the product: onboard once, create projects, launch workloads, then manage cost, security, and operations from the same workspace.

## Start here

<Steps>
  <Step title="Create or select an organization">
    Use the [organization guide](/console/organizations) to define the workspace that owns billing, members, and shared policy.
  </Step>
  <Step title="Create a project">
    Projects are the unit Nubis uses for scoping workloads, networking, and cost visibility. Start with one project per environment or product.
  </Step>
  <Step title="Add billing and verification">
    Set a payment method, confirm any required billing checks, and decide on spend controls before you launch production workloads.
  </Step>
  <Step title="Launch resources">
    Provision your first [instance](/services/instances/create), database, Kubernetes cluster, bucket, or DNS zone from the Console.
  </Step>
</Steps>
